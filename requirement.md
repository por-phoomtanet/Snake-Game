# 🐍 TikTok Snake Live — เอกสารข้อกำหนด (Requirements Specification)

| | |
|---|---|
| **โครงการ** | เกมงู AI สำหรับไลฟ์ TikTok รองรับของขวัญ |
| **เวอร์ชัน** | 1.0 (Phase 1 เสร็จ) |
| **วันที่** | 2026-07-01 |
| **สถานะ** | Phase 1 ใช้งานได้ · Phase 2 วางโครงแล้ว |

---

## 1. ภาพรวม & เป้าหมาย
เกมงูที่ **AI เล่นเอง** ต่อเนื่องได้ตลอดการไลฟ์ TikTok โดยไม่น่าเบื่อ ผู้ชมมีส่วนร่วมผ่าน
**การส่งของขวัญ** ที่เปลี่ยนแปลงเกมทันที (ชื่องู/สกิน/กำแพง/ธีม) และค่อยๆ ยากขึ้นเรื่อยๆ
เฟสถัดไปเพิ่ม **พิธีกร AI** พูดขอบคุณของขวัญและตอบแชตได้

**กลุ่มผู้ใช้**
- **สตรีมเมอร์** — รันเกมเป็น overlay ใน OBS, จูนค่าต่างๆ
- **ผู้ชม TikTok** — ส่งของขวัญ/แชตเพื่อมีอิทธิพลต่อเกม

---

## 2. การตัดสินใจด้านสถาปัตยกรรม (ยืนยันแล้ว 2026-07-01)
| หัวข้อ | ทางเลือกที่เลือก | เหตุผล |
|--------|-----------------|--------|
| โหมดเล่น | **บอท AI เล่นเอง** | ไลฟ์ต่อเนื่องได้เองไม่ต้องคุม |
| แพลตฟอร์ม | **เว็บ (HTML5 Canvas) + OBS overlay** | เข้า OBS เป็น Browser Source ได้ทันที ทำ overlay ใสง่าย |
| เชื่อม TikTok | **เริ่มด้วยตัวจำลอง** → ต่อ TikTok-Live-Connector ภายหลัง | ทดสอบ/พัฒนาได้โดยไม่ต้องไลฟ์จริง |
| ผลของขวัญ | ตั้งชื่องู+สกินตามคนส่ง / กำแพงมีชื่อคนส่ง / เปลี่ยนธีม | ตามที่ผู้ใช้ระบุ |

## 3. ข้อกำหนดเชิงฟังก์ชัน (Functional Requirements)

### 3.1 เกมหลัก (Core Gameplay)
| ID | ข้อกำหนด | สถานะ |
|----|----------|-------|
| FR-1.1 | งูเคลื่อนที่อัตโนมัติด้วย AI (หาเส้นทางไปอาหารด้วย BFS) | ✅ |
| FR-1.2 | AI ต้องหลีกเลี่ยงการขังตัวเองด้วย flood-fill ก่อนตัดสินใจแต่ละก้าว | ✅ |
| FR-1.3 | เมื่อไม่มีทางปลอดภัยไปอาหาร → เข้าโหมดเอาตัวรอด (เดินไปที่ว่างมากสุด) | ✅ |
| FR-1.4 | กินอาหารแล้วงูยาวขึ้น +คะแนน และสร้างอาหารใหม่ในช่องว่าง | ✅ |
| FR-1.5 | ชนขอบ/ตัวเอง/สิ่งกีดขวาง/กำแพง → ตาย แล้วเกิดใหม่ทันที (หักคะแนน) | ✅ |
| FR-1.6 | เล่นต่อเนื่องไม่มีหน้าจอ Game Over ค้าง (เหมาะกับไลฟ์) | ✅ |

### 3.2 ความยาก / อัพเกรด (Progression)
| ID | ข้อกำหนด | สถานะ |
|----|----------|-------|
| FR-2.1 | ขึ้นด่านทุก N อาหาร (ค่าเริ่มต้น 4) | ✅ |
| FR-2.2 | ทุกด่านที่ขึ้น → งูเร็วขึ้น (ลด tick) จนถึงเพดานความเร็ว | ✅ |
| FR-2.3 | ทุกด่านที่ขึ้น → เพิ่มสิ่งกีดขวางถาวร (จำกัดเพดานกันแน่นเกิน) | ✅ |
| FR-2.4 | แสดงเอฟเฟกต์ "LEVEL X ⚡" เมื่อขึ้นด่าน | ✅ |

### 3.3 ระบบของขวัญ (Gifts) — หัวใจของการมีส่วนร่วม
| ID | ข้อกำหนด | สถานะ |
|----|----------|-------|
| FR-3.1 | **ทุกของขวัญ** → งูเปลี่ยนชื่อเป็นชื่อคนส่ง + เปลี่ยนสกิน | ✅ |
| FR-3.2 | เวลาแสดงสกิน = `coins × qty` วินาที (ขั้นต่ำ 8s / สูงสุด 600s) | ✅ |
| FR-3.3 | คนส่งหลายคน → เข้า **คิว** สลับกันเป็นเจ้าของงู ทุกคนได้โชว์ชื่อ | ✅ |
| FR-3.4 | ของขวัญประเภท **กำแพง** → สร้างกำแพงมีป้ายชื่อคนส่ง | ✅ |
| FR-3.5 | เวลากำแพงอยู่ = `WALL_SEC_PER_QTY × qty` วินาที แล้วหายเอง | ✅ |
| FR-3.6 | AI ต้องหลบกำแพงที่ผู้ชมสร้าง (ถือเป็นสิ่งกีดขวาง) | ✅ |
| FR-3.7 | ของขวัญประเภท **ธีม** → เปลี่ยนธีม/สีทั้งจอ | ✅ |
| FR-3.8 | ของขวัญใหญ่ (epic เช่น Lion) → สกินรุ้ง + มงกุฎ + คอนเฟตติ | ✅ |
| FR-3.9 | แคตตาล็อกของขวัญ/สกิน/ธีม แก้ไข-เพิ่มได้จากไฟล์เดียว (`config.js`) | ✅ |

## 4. ข้อกำหนดที่ไม่ใช่ฟังก์ชัน (Non-Functional)
| ID | ข้อกำหนด |
|----|----------|
| NFR-1 | เปิด `index.html` เล่นได้ทันที ไม่ต้องติดตั้ง/build (Phase 1) |

| NFR-5 | **ความปลอดภัย:** ห้ามเก็บ `ANTHROPIC_API_KEY` ฝั่งเบราว์เซอร์ — เก็บใน backend เท่านั้น |
| NFR-6 | รองรับ overlay แบบพื้นหลังใสใน OBS ขนาด 960×540 (ปรับได้) |

## 5. สถาปัตยกรรม & โครงสร้างไฟล์
```
index.html          หน้าเกม (โหลดใน OBS ได้เลย)
styles.css          สไตล์ overlay สำหรับไลฟ์
js/config.js        ⭐ จูนทุกอย่าง + แคตตาล็อกของขวัญ/สกิน/ธีม
js/events.js        Event Bus กลาง (แยกแหล่งข้อมูลออกจากเกม)
js/ai.js            สมองงู (BFS + flood-fill กันขดตาย)
js/game.js          เครื่องยนต์เกม (สถานะ/เดิน/ด่าน/กำแพง/คิวเจ้าของ)
js/gifts.js         ของขวัญ → เอฟเฟกต์
js/render.js        วาดจอ + HUD/ทอสต์/ลีดเดอร์บอร์ด
js/simulator.js     ตัวจำลองของขวัญ/แชต (ทดสอบ)
js/aihost.js        พิธีกร AI (TTS + จุดต่อ Claude)
js/main.js          บูต + ลูปหลัก
server/             โครง backend เฟส 2 (TikTok จริง + Claude)
```

**การไหลของข้อมูล (แยกส่วนชัดเจน):**
```
แหล่งข้อมูล ─┐
  Simulator │ emit('gift'/'chat')
  TikTok WS ─┘        ↓
              [ SG.Bus (Event Bus) ]
                 ↓         ↓        ↓
              gifts.js  aihost.js  render(toast/chat)
                 ↓
              game.js state → render.js วาดจอ
```

## 6. รูปแบบ Event มาตรฐาน (สัญญาระหว่างแหล่งข้อมูลกับเกม)
```js
SG.Bus.emit('gift', { user: string, giftId: string, qty: number });
SG.Bus.emit('chat', { user: string, message: string });
```
> ต่อ TikTok จริง: map `giftId` ของ TikTok → คีย์ใน `SG.GIFTS` แล้ว emit เข้า Bus เดิม

## 7. ค่าปรับแต่งหลัก (ค่าเริ่มต้น — แก้ที่ `js/config.js`)
| ค่า | ความหมาย | ค่าเริ่มต้น |
|-----|----------|------------|
| `grid` | ขนาดกระดาน (คอลัมน์×แถว×ช่อง) | 32×18×30 |
| `baseTickMs` / `minTickMs` | ความเร็วเริ่ม / เร็วสุด | 130 / 55 ms |
| `speedUpPerLevel` | เร็วขึ้นต่อด่าน | 6 ms |
| `foodPerLevel` | อาหารต่อ 1 ด่าน | 4 |
| `SKIN_SEC_PER_COIN` / min / max | สูตรเวลาสกิน | 1 / 8s / 600s |
| `WALL_SEC_PER_QTY` | เวลากำแพงต่อชิ้น | 12s |
| `maxOwnerQueue` | ความยาวคิวเจ้าของงู | 12 |

## 8. แผนงาน (Roadmap)
- **Phase 1 (เสร็จ)** — เกม AI + ระบบของขวัญครบ + overlay + ตัวจำลอง + TTS ขอบคุณ
- **Phase 2** — ต่อ TikTok LIVE จริง (WebSocket) + ตอบแชตด้วย Claude (`claude-haiku-4-5`) ผ่าน backend
- **แนวคิดต่อยอด** — เสียงเอฟเฟกต์, มินิบอสประจำด่าน, โหวตทิศทางด้วยแชต, สถิติสรุปท้ายไลฟ์
